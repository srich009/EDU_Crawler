UCR's Resource for Information to Access Campus Enterprise Systems

UCR has many computer systems available to Faculty and Staff to assist them in conducting campus business. The computer systems range from those that can be accessed via the web or those that require software to be installed on your computer.

Access to campus systems is authorized by the System Access Administrator (SAA) via The Enterprise Access Control System (EACS). 

EACS is a companion system to the campus Enterprise Directory. The Enterprise Directory provides a common identifier known as a UCR NetID that is used to authenticate individuals to various UCR electronic systems and services.   The Enterprise Directory Administrator assigns employees a UCR NetID and password, while EACS determines whether or not individual users have the authority to access campus systems and applications.

Employees access systems, for which they have been authorized by the SAA, from UCR’s single-sign on portal environment, R’Space via the Authorized Applications Portlet.  The majority of enterprise systems now utilize this portal environment but there are some applications that must be accessed directly from a specific web page or desktop application.

This web page was developed to provide guidance to SAAs to help in determining which applications are recommended for authorization as well as to provide details regarding each application.  Details include a brief description of the application, training requirements as well as support and user guide resources. 

Please note, several systems (e.g., UCR Financial System, Payroll and Personnel System and the Student Data Query System) require online and/or classroom training before access can be authorized.  These details will be included in the summary section of the application.

Please refer to the All Applications section for a complete listing of campus applications.  There are also links available from the left menu for a listing of suggested applications by type of employee.

As a final note, please keep in mind that several systems require an IBM account and a multi-step process to obtain access. For example, access to SIS+ (Student Information System) requires an IBM account and the privileges are assigned by the Registrar's Office.  This information will be included in the summary section of the application.
